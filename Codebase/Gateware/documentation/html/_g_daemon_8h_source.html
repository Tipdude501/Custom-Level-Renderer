76  //! Allocates & Intializes a GDaemon.
77  /*!
78  * The GDaemon is used by end users and even Gateware itself to schedule repeating background thread work.
79  * Even though you can create multiple instances of a GDaemon they all utilize the same background thread pool.
80  * While simple recuring tasks based on the passage of time are standard, you can also create co-dependent
81  * chains of tasks by using the built-in events to "Pause" and "Resume" other GDaemons.
82  *
83  * \param [in] _targetInterval How often (MILLISECONDS) should this daemon attempt to execute the operation provided.(inclusive)
84  * Recurring tasks are launched based on an absolute time interval rather than a relative time interval.
85  * NOTE: If an operation takes longer than the target interval, the next operation will be delayed.
86  * \param [in] _daemonOperation (OPTIONAL) The function to be executed recurring based on the provided interval.
87  * NOTE: If an operation is not provided the Daemon will still spawn an events at the interval.
88  *
89  * \retval GReturn::INVALID_ARGUMENT The _targetInterval must be greater than 0 milliseconds.
90  * \retval GReturn::SUCCESS The Daemon was successfully intialized and created and is now running.
91  */
92  GReturn Create(unsigned int _targetInterval, std::function<void()> _daemonOperation);

133  //! Requests that a GDaemon cease re-scheduling of operations once/if the current operation is complete.
134  /*!
135  * If an operation is currently active the pause will take effect once it completes.
136  * The Pause will not block the current thread of execution unless "_wait" is true.
137  *
138  * \param [in] _wait Should this function block until the active execution of this Daemon has completed?
139  * NOTE: Do not set this to true if you are on the same thread as the Daemon (ex: within Operation).
140  * \param [in] _spinUntil Determine how long this thread will spinlock waiting for daemon to complete (NANOSECONDS).
141  * Once "_spinUntil" nanoseconds has elapsed the current thread will yield to the OS scheduler.
142  * NOTE: If your not in a time/performance critical area you should just use 0.
143  *
144  * \retval GReturn::DEADLOCK You have attempted wait for Daemon within itself. (This would cause a Deadlock)
145  * \retval GReturn::SUCCESS We have successfully signaled the Daemon to hold off on future operations.
146  * \retval GReturn::REDUNDANT Future scheduled Operations have already been Paused.
147  */
148  virtual GReturn Pause(bool _wait, unsigned int _spinUntil) = 0;
