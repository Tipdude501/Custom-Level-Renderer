129  //! Launch a multi-thread spanning simulatenous array modification operation to run concurrently with this thread.
130  /*!
131  * Allows you to process large contigious data sets using a divide & conquer approach.
132  * Based on how many "_maxSections" are required to traverse "_arraySize" we create that many jobs in the internal thread pool.
133  * Then each job will run "_parallelTask" for each input/output pair in it's designated section (optional array location is provided).
134  * To know when a single section completes listen for Events::PARALLEL_SECTION_COMPLETE. There is no cross section ordering garuntee.
135  * Global resources accessed by the provided operation are your responsibility to synchronize for thread safety (recc: GThreadShared).
136  * However the provided arrays are safely accessed due to being divided evenly amongst the internal jobs (no overlap).
137  * You may wait for this entire operation to complete using "Converge" or listen for Events::PARALLEL_TASK_COMPLETE
138  * Once you have "Converged" or received the appropriate message(s) you are safe to acd amongst processing cores.
139  *
140  * \param [in] _parallelTask Pointer to static operation responsible for element processing, called once for each specific element.
141  * \param [in] _maxSection The maximum # of elements processed by a thread at once, this has the largest impact on perfomance.
142  * NOTE: As rule of thumb this value should be smaller for complex operations and larger for simplistic ones.
143  * \param [in] _arraySize The size of array or problem domain that is to be divideey wish each routine to have access to.
144  * \param [in] _userData (OPTIONAL) Pointer to some custom user provided data that the input/output data arrays might need.
145  * \param [in] _inStride (OPTIONAL) if 0 then sizeof(Input) type used for input array traversal, otherwise this byte amount is used.
146  * \param [in] _inputArray (OPTIONAL) Array of input data elements that will be fed to the processing system.
147  * \param [in] _outStride (OPTIONAL) if 0 then sizeof(Output) type used for output array traversal, otherwise this byte amount is used.
148  * \param [out] _outputArray (OPTIONAL) Array of output data elements that will be fed to the processing system.
149  *
150  * \retval GReturn::MEMORY_CORRUPTION _inputArray & _outputArray have an overlapping memory space. (Just use _outputArray instead)
151  * \retval GReturn::INVALID_ARGUMENT You must provide a non-null function to execute and a non-zero arraySize & maxSection.
152  * Cannot use the same array for _inputArray and _outputArray. In this situation, just use _outputArray.
153  * \retval GReturn::SUCCESS We have successfully submitted the task for processing.
154  */
155  template<typename Input, typename Output>
156  GReturn BranchParallel(void(*_parallelTask)(const Input*, Output*, unsigned int, const void*), unsigned int _maxSection, unsigned int _arraySize, const void* _userData, int _inStride, const Input* _inputArray, int _outStride, Output* _outputArray);

158  //! Forces the current thread to wait until any & all branched processing has completed.
159  /*!
160  * Polls the GConcurrent until all running tasks have completed.
161  * Use this when you need to access resources on the current thread that may still be processing elsewhere.
162  * If you want a less invasive method of waiting and responding, consider using the built-in messaging system to be notified instead.
163  *
164  * \param [in] _spinUntil Determine how long the current thread will spinlock waiting for all Tasks to complete. (in NANOSECONDS)
165  * Once "_spinUntil" nanoseconds has elapsed the current thread will yield to the OS scheduler. (1 millisecond yield)
166  * Upon being re-scheduled the thread will try to spinlock again until "_spinUntil" is reached again or completion.
167  * NOTE: If your not in a time/performance critical area you should just use 0. Wake times dependent on OS resolution.
168  *
169  * \retval GReturn::FAILURE There have been no Tasks ever or since you last Converged.
170  * \retval GReturn::SUCCESS Tasks were running and we sucessfully waited for their completion.
171  */
172  virtual GReturn Converge(unsigned int _spinUntil) = 0;
