94  /*!
95  * A GEventQueue is capable of receiving and reacting to events created by a GEventGenerator.
96  * You can use this class whenever there is a Gateware class that spawns events you wish to know about.
97  * Unlike GEventReceiver the GEventQueue is capable of storing multiple events up to a limit defined by you.
98  * If you need to process events on a specific thread, you can use this class to delay processing until
99  * a safe time to do so. Just make sure there is adequate capcity in the Queue, and "Pop" them off when ready.
100  *
101  * \param [in] _maxSize The maximum amount of messages that can be retained by this instance. (must be > 0)
102  * Exceeding this amount without processing a message will cause the oldest to be missed.
103  * \param [in] _eventSource The GEventGenerator you wish to receive messages from.
104  * \param [in] _callback The function or lambda you wish to be invoked when an event is received.
105  * (optional, can pass nullptr). May be invoked from a different thread.
106  * To ensure you receive messages on a specific thread, just use "Pop/Peek" instead.
107  *
108  * \retval GReturn::UNEXPECTED_RESULT Internal API failure, should not happen please report this.
109  * \retval GReturn::INVALID_ARGUMENT A valid GEventGenerator and Queue Capacity > 0 must be provided.
110  * \retval GReturn::SUCCESS The receiver was correctly bound and registered to the generator provided.
111  */
